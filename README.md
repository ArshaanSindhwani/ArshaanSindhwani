### AgroSense Architecture
- Hybrid offline-compatible model using local persistence + cloud sync  
- Queue-based synchronisation system for deferred network operations  
- Firestore as centralised backend data store  
- Event-driven UI updates based on connectivity state  

### DataOps Pipeline Architecture
- Scheduled GitHub Actions workflow for automated ingestion  
- Stateless Node.js transformation layer for data processing  
- Docker containerisation for reproducible execution environments  
- Dual-format persistence model (JSON + CSV logging)  
- CI/CD deployment pipeline triggered via Render hooks  

### Backend Engineering Practices
- Layered architecture (controller → service → data model separation)  
- JWT-based authentication and route protection  
- Input validation and structured error handling  
- Unit + integration testing using Jest and Supertest  

### DevOps Practices

- Containerised deployments using Docker
